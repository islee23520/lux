# Workflow Skill Routing

Use this category for LUX runtime workflows and game-development control-plane operations.

## Route By Intent

- `game-dev`: use for compact game planning, core loop validation, prototype scope, QA plan, ethics, and context-first engine observation.
- `lux-unity`: use for LUX Unity bridge and MCP operations: status, compile, tests, logs, launch, screenshots, hierarchy, dynamic code, record/replay, and one-loop MCP workflows.

## Engine Maturity

Unity is the verified path. Godot and Three.js must be routed through explicit capability status and blocker evidence. Do not claim Unreal, 3ds, Godot, or Three.js parity unless a local adapter and verification surface prove it.
