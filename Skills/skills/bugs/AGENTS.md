# Bug And Maintenance Skill Routing

Use this category for defects, maintenance records, release notes, and postmortems.

## Route By Intent

- `bug-report`: use to turn an observed failure into an actionable report with environment, steps, expected result, actual result, logs, severity, and reproducibility.
- `bug-triage`: use to classify and route defects before implementation.
- `tech-debt`: use to document or reduce debt with affected files, invariant risk, simplest cleanup, and tests needed.
- `retrospective`: use after incidents or milestones.
- `changelog`: use to write release notes from verified changes.

## Do Not Use For

- Security-specific findings. Use `../review/security-audit`.
- Release readiness gates. Use `../quality/release-checklist`.
