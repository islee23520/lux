# Unity Skill Routing

Use this category for Unity-specific implementation reference and design patterns.

## Route By Intent

- `unity-cs-reference`: use for Unity Editor API decisions, bridge automation, AssetDatabase, BuildPipeline, CompilationPipeline, EditorApplication, UI Toolkit, PackageManager, serialization, and ScriptableObject persistence.
- `patterns/unity-pattern-*`: use only when a Unity/C# design pattern is directly relevant to implementation or review.

## Pattern Selection

Open the specific pattern skill, not every pattern. Pick the narrowest matching pattern from `patterns/` and keep it as reference guidance, not a requirement to over-abstract.
